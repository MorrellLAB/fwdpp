1 # Example programs
2 
3 The best documentation of how to use the library are the example simulations in the examples/ subdirectory.
4 
5 The individual-based programs represent the best/most efficient way to use __fwdpp__. The others are here for legacy purposes.
6 
7 ###Compiling the examples:
8 
9 As of fwdpp 0.2.5, the examples are compiled by saying "make check" after the configure step.
10 
11 ###Running the examples:
12 
13 This following list of parameters and their definitions is in common to all of the example programs:<br>
14 N = the number of diploid individuals to simulate<br>
15 theta = 4Nu, where u is the mutation rate per gamete per generation. For example, theta = 100 means that on average, 2Nu = 50 new mutations are entering the population each generation.<br>
16 rho = 4Nr, where r is the recombination rate per diploid per generation<br>
17 g = the number of generations to simulate. Often, this should be >= 8N at a minimum<br>
18 n = the sample size to draw at the end of the simulation. To match the typical modeling assumptions of population genetics, you should have n << 2N.<br>
19 nreps = the number of replicates to simulate<br>
20 seed = a random number seed. I will use $RANDOM as a seed, referring to the bash shell method to return a random integer.<br>
21 
22 Note: familiarity with Hudson's "[ms](http://home.uchicago.edu/~rhudson1)" program is helpful for some of what comes below.
23 
24 ####diploid_ind (diploid_ind.cc)
25 
26 The first program, diploid, simulates a Wright-Fisher population with mutation, recombination, and drift. To run it:
27 
28 ~~~
29 ./diploid_ind N theta rho g n nreps seed | gzip > outfile.gz
30 ~~~
31 
32 The data in outfile.gz will be in the same format as Dick Hudson's "ms" coalescent simulator.
33 
34 Example:
35 
36 ~~~
37 ./diploid_ind 10000 10 10 100000 50 1 $RANDOM | gzip > test_diploid.gz
38 ~~~

41 ####diploid\_binaryIO_ind (diploid_binaryIO_ind.cc)
42 
43 The next program is called diploid\_binaryIO\_ind. This program is identical to diploid, except that it only simulates one replicate at a time, creates two output files, and outputs the entire population rather than a sample of size n << 2N. The first file is an index file, containing an integer representing the replicate number of the output and the position in the haplotypes file where this record begins. The second file is the haplotypes file, which contains the entire population in binary format.
44 
45 Usage:
46 
47 ~~~
48 ./diploid_binaryIO N theta rho g replicate\_number index\_filename haplotype\_filename seed.
49 ~~~
50 
51 Example for Open Grid Engine compute clusters:
52 
53 ~~~{.sh}
54 #!sh
55 #$ -t 1-100
56 #$ -N DIPBIN
57 
58 seed=`echo "$SGE_TASK_ID*$RANDOM"|bc -l`
59 ~~~
60 
61 \#note: the below assumes that the binary is in the users's $PATH and that the GE system knows how to link it to the relevant dynamic libraries
62 
63 ~~~
64 diploid_binaryIO_ind 10000 10 10 100000 $SGE_TASK_ID indexfile hapfile $seed
65 ~~~
66 
67 The above script, when submitted to a Grid Engine queue, will result in 100 populations of size N=10,000 being written to hapfile. Further, “indexfile” will contain the ID number and position of each file. Records are not over-written because the program uses POSIX file locking to ensure that only 1 process at a time can do the writing. This is a complex program, as it mixes C++ objects with output streams such that they can be written to C-style file descriptors, which is required in order to use file locking (which is a C feature with no C++ analog). However, the advantage is that you write all data to one large file, avoiding the plague of lots of small files that can bring distributed file systems to their knees.

71 
72 This program is similar to diploid_ind, but adds an additional mutation rate (theta\_selected = 4Nu\_s, where u\_s is the mutation rate per gamete per generation to mutations with selection coefficient s) to mutations with selection coefficient s and dominance h. Fitness across sites is multiplicative. The output is in "ms" format--one block for neutral mutations followed by one block for selected mutations.
73 
74 Usage:
75 
76 ~~~
77 ./dipoid_fixed_sh_ind N theta theta_selected rho s h g n nreps seed | gzip > outfile.gz
78 ~~~
79 
80 For this program, s can be positive or negative, as can h.

82 ####diploid\_twopop\_mig\_ind (diploid_twopop_mig_ind.cc)
83 
84 This program simulates an ancestral population for g generations, at which point a daughter population of size N is “budded” off from the ancestral population. Evolution continues for g2 more generations, with symmetric migration at rate M = 4Nm, where m is the migration rate per diploid per generation. The output is in "ms" format, with n haplotypes per sample just like how ms outputs data for multi-population models.
85 
86 Usage:
87 
88 ~~~
89 ./diploid_twopop_mig_ind N theta rho g g2 M n nreps seed | gzip > outfile.gz
90 ~~~
91 
92 Note: the demographic model here implemented may be viewed as biologically bizarre, as it mimics the default behavior of “ms” for population split models. Let’s compare a specific example vs. the equivalent ms command line:
93 
94 ~~~
95 ./diploid_twopop_mig 10000 50 50 100000 1000 1 50 1 $RANDOM | gzip > outfile.gz
96 ~~~
97 
98 and
99 
100 ~~~
101 ms 100 1 -t 50 -r 50 1000 -I 2 2 1 -ej 0.025 2 1 -em 0.025 1 2 0.
102 ~~~
103 
104 Why may this be considered odd? In ms, when two populations are merged, the rate of coalescence is unaffected by default (this behavior is documented, and it is up to the user to adjust population sizes when population merge and split in ms). That means when the two populations, each of size N merge, the merged (ancestral) population is still of size N. diploid\_twopop\_mig is doing the same thing forwards in time: an ancestral population of size N magically changes into two populations of size N.

106 ####migsel\_ind (migsel_ind.cc)
107 
108 Simulates 2 equal-sized populations of size N (N remains constant over time) diploids with selection at strength s and dominance h. Migration occurs between the two populations.
109 
110 _Note that the two populations never share a common ancestor in this simulation! In other words, deme 1 is never founded by a sampling event from deme 0. This means that if you simulate for a short period of time, it is unlikely that there is a MRCA common to all individuals sampled. (See comment above about appropriateness of these examples for research...)_
111 
112 Usage:
113 
114 ~~~
115 ./migsel_ind N 4Nu_neut 4Nu_sel 4Nr 4Nm s h f1 f2 ngens n outfilename seed
116 ~~~
117 
118 where:
119 N = population number for each deme.
120 4Nu\_neut = 4N times the neutral mutation rate per gamete
121 4Nu\_sel = 4N times the mutation rate per gamete to selected mutations
122 4Nr = 4N times the recombination rate per diploid per region per generation
123 4Nm = 4N times the probability of migration per diploid per generation
124 s = the selection coefficient of newly-arising selected mutations
125 h = dominance of newly-arising selected mutations
126 f1 and f2 are the probabilities of inbreeding in deme 1 and 2, respectively
127 ngens = # of generations to simulate
128 n = sample size to take from each deme @ end of simulation (must be even #)
129 outfilename = name of output file
130 seed = random number seed.
131 
132 Notes: s is taken to be s in deme 1 and -s in deme 2. This lets me illustrate how different fitness functions can be passed to different demes using fwdpp.
133 
134 The output file contains the following:
135 
136 1. The metapopulation in binary format
137 2. Two "ms"-format blocks of size 2*n. These are for neutral and selected mutations, respectively. Within each block, the first n haplotypes are from deme 1 and the second n are from deme 2.
138 
139 The program writes the data to the output file and then reads it in again. This is mainly to illustrate the binary I/O routines for individual-based metapopulation simulations.

179 ####bneck\_selection\_ind (bneck_selection_ind.cc)
180 
181 This program simulates a population for g generations at size N. In generation g+1, N changes to N2 <= N. The population then grows exponentially to size N3 >= N2 in g2 generations. Selected and neutral mutations are allowed each generation. The output is in “ms” format--one block for neutral mutations followed by one block for selected mutations.
182 
183 Usage:
184 
185 ~~~
186 ./bneck_selection_ind N theta_neutral theta_sel rho s h g1 N2 N3 g2 n nreps seed
187 ~~~
188 
189 Where:<br>
190 N = starting population size<br>
191 theta\_neutral = 4N*(neutral mutation rate per gamete)<br>
192 theta\_sel = 4N*(mutation rate per gamete to selected mutations)<br>
193 rho = 4Nr, where r is recombination rate per diploid per generation<br>
194 s = selection coefficient. Can be negative or positive.<br>
195 h = dominance of selected mutations<br>
196 N2 = size of bottlenecked population<br>
197 N3 = size of recovered population<br>
198 g2 = generations taken to go from size N2 to size N3<br>
199 n = sample size to take from the population.<br>
200 nreps = # replicates to simulate<br>
201 seed = random number seed.<br>

203 ####diploid\_ind\_2locus (diploid_ind_2locus.cc)
204 
205 This program simulates a 2-locus neutral model. Mutations arise at rate \f$\theta\f$ at each locus, and each locus recombines at rate \f$\rho\f$. The recombination rate between loci is \f$r_{bw}\f$, which corresponds to the probability a crossover is observed between loci. In other words, the genotypes at the two loci will switch from \f[\frac{AB}{ab}\f] to \f[\frac{Ab}{aB}\f] with probability \f$r_{bw}\f$.
206 
207 The output of the simulation is an "ms"-style block for each locus. Positions are uniform on the interval \f$[0,1)\f$ at locus 1, and uniform on the interval \f$[1,2)\f$ at locus 2.
208 
209 The usage is:
210 
211 ~~~
212 Usage:
213 ./diploid_ind_2locus N theta rho rbw ngens n nreps seed
214 Where:
215 N = population size (number of diploids)
216 theta = 4Nu, the scaled neutral mutation rate
217 rho = 4Nr, the scale recombination rate
218 rbw = the probability that the two loci cross over, per generation
219 ngens = the number of generations to simulate
220 n = the sample size to pull from the population at the end of each simulated replicate
221 nreps = the number of replicates to simulated
222 seed = seed value for random number generations
223 ~~~

225 #### HOC_ind (HOC_ind.cc)
226 
227 This program simulates a little twist on Turelli's House-of-Cards (HOC) model. The main point of this model is to demonstrate how to use KTfwd::tags::gamete_dependent to implement mutation models that depend on the current gamete state.
228 
229 In a typical HOC model (Kingman), the effect size of a mutated allele is a new draw from a Gaussian distribution. In this simulation, the current effect size of an allele, before mutation, is additive over causative mutations, with sum \f$y\f$. After the addition of a single mutation, its new effect size will be \f$x\f$, and thus the effect size of the new mutation, \f$e\f$, is chosen such that \f$x = y+e\f$.
230 
231 The fitness model here is Gaussian stabilizing selection with a mean of 0 and a standard deviation of 1. With this fitness function and the additive model of Turelli, \f$V(G) \approx 4\mu\f$ when the ratio of the variance in effect sizes over the variance in fitness falls within a certain range. That approximation seems to hold, at least roughly, for this odd method of generating the effect sizes of new variants. For example, these commands return a mean of approximately \f$4 \times 10^{-4}\f$
232 
233 ~~~
234 #Generate 100 seeds
235 Rscript -e "cat(as.integer(runif(1e2,0,1e6)),sep=\"\n\")" > seeds
236 #run jobs in parallel on a 4-core machine, write VG to a file
237 parallel --jobs 4 ./HOC_ind 10000 0.0001 0.25 0 100000 1 {} :::: seeds > VG.txt
238 ~~~
239 
240 The usage is:
241 
242 ~~~
243 Usage: ./HOC_ind N mu sigmu rho ngens nreps seed
244 where:
245 N = diploid population size
246 mu = mutation rate to variants affecting fitness
247 sigmu = standard deviation of effect sizes. E ~ N(0,sigmu^2)
248 4Nr = population scaled recombination rate (per diploid, per generation)
249 ngens = number of generations to simulate
250 nreps = number of replicates to simulate
251 seed = random number seed
252 ~~~
253 
254 The only output is \f$V(G)\f$, the genetic variance in fitness, which is written to stdout for each replicate.
