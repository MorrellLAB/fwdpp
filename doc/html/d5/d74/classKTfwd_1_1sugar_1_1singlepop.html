Detailed Description

template<typename mutation_type, typename mlist, typename glist, typename dipvector, typename mvector, typename ftvector, typename lookup_table_type>
class KTfwd::sugar::singlepop< mutation_type, mlist, glist, dipvector, mvector, ftvector, lookup_table_type >

Abstraction of what is needed to simulate a single population using an individual-based sampler from fwdpp.

All that is missing is the mutation_type and the container types.

Does not allow copy construction/assignment!

See Tutorial 5: the "sugar" layer on top of fwdpp for rationale, etc.

Examples:
bneck_selection_ind.cc, diploid_fixed_sh_ind.cc, diploid_fixed_sh_ind_lambda.cc, diploid_ind.cc, and migsel_ind.cc.

template<typename mutation_type , typename mlist , typename glist , typename dipvector , typename mvector , typename ftvector , typename lookup_table_type >
gamete_t::mutation_container KTfwd::sugar::singlepop< mutation_type, mlist, glist, dipvector, mvector, ftvector, lookup_table_type >::neutral

Vectors for holding copies of pointers to mutations during recombination. The requirement to declare these was introduced in fwdpp 0.3.3.

In previous versions of the library, vectors like this had to be allocated for every crossover event for every generation. The result was an excessive number of requests for memory allocation.

Now, we create the vector once per replicate. Further, we will reserve memory here, to minimize reallocs, etc., within fwdpp.

Internally, fwdpp's job is to make sure that this vector is appropriately and efficiently cleared, but only when needed.

Note
: if not using the sugar features, you can create these vectors only once per simulation...
Examples:
bneck_selection_ind.cc, diploid_fixed_sh_ind.cc, diploid_fixed_sh_ind_lambda.cc, and diploid_ind.cc.
