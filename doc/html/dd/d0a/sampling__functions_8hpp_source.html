14 /*! @defgroup samplingPops Functions related to taking samples from simulated populations
15 
16  This collection of functions allows a user to draw a sample of size \f$n \ll 2N\f$ from
17  a simulated population.
18 
19  The library provides several overloads of the functions KTfwd::ms_sample and KTfwd::ms_sample_separate.
20 
21  The following features are in common to all versions of these functions:
22 
23  1. They only supp]ort bi-allelic mutation positions.
24  2. All functions return some number of objects of type std::vector< std::pair<double, std::string> >.
25  Each element in the vector corresponds to a mutation. The double corresponds to the mutation position,
26  and the string corresponds to the character states in each of the \f$n\f$ samples. The strings contain the character
27  '0' when a sample has the ancestral state or '1' for the derived (mutant) state. This type may be used
28  to populate a polymorphism table from the [libsequence](http://molpopgen.github.io/libsequence/) library:
29  \code
30  auto x = KTfwd::ms_sample( appropriate arguments );
31  Sequence::SimData xx(x.begin(),x.end());
32  \endcode
33  When a SimData object is written to a stream, its output format is the same as that used by Dick Hudson's coalescent simulation
34  program [ms](http://home.uchicago.edu/~rhudson1/source/mksamples.html)
35  3. All versions of KTfwd::ms_sample return vectors where the mutations affecting fitness and those not affecting fitness are intermingled.
36  4. All versions of KTfwd::ms_sample_separate return pairs of vectors separating the mutations not affecting fitness from those that do.
37  The first member of each pair is a vector of "neutral" mutations, and the second member is the vector of "selected" mutations.
38  5. The functions are all implemented using sampling with replacement from the population. Thus, setting \f$n = N\f$ (or \f$2N\f$) will NOT
39  return all the variable sites in the entire population!!!
40 
41  A vector may contain data looking like the following:
42  \verbatim
43  0.0760 "10000"
44  0.6805 "00010"
45  \endverbatim
46 
47  The two rows are the two different positions (0.076 and 0.6805). The first haplotype is "10", the second is "00", and the fourth is "01".
48 
49  If you converted the data to a Sequence::SimData object and printed it to screen or a file, the output would be:
50 
51  \verbatim
52  //
53  segsites: 2
54  positions: 0.0760 0.6805
55  10
56  00
57  00
58  01
59  00
60  \endverbatim
61 */

63 /*!
64  @defgroup samplingPopsGamete Randomly-sampling individual gametes.
65  @ingroup samplingPops
66 
67  These functions draws a sample of size \f$n \ll 2N\f$ from a simulated population.
68 
69  The gametes are randomly-sampled (with replacement) proportionally to their frequency in the population.
70 
71  The object passed to these functions is the container of gametes (e.g, some container of type KTfwd::gamete_base).
72 
73  The return values have no relation to any actual diploid individual in the population. Each haplotype
74  corresponds to a simulated haplotype, but adjacent haplotypes are basically random draws of M&Ms from a jar.
75 
76  For these functions, \f$n\f$ can be odd or even.
77 */
