16  This type represents a mutation with a number of 's' and 'h' parameters that are fixed
17  at compile time.
18 
19  For example, generalmut<2> could represent separate effects on fitness and a trait, while
20  generalmut<4> could represent separate effects on fitness and on a trait in males and females.
21 
22  Other example use cases could be generalmut<N> representing fitness/dominance value pairs in
23  different 'patches' of an environment.
24 
25  The advanatage of this type is that std::array<double,N> is a very fast type (often faster
26  than a std::vector<double> of the same size). Further, this type can be filled via compile-time
27  meta-programming techniques.
28 
29  The disadvantage of this type is that it is not easy to have the number of 's,h' pairs be flexible
30  at run-time. For that use case, see KTfwd::generalmut_vec, which uses std::vector<double> instead of
31  std::array to store the 's,h' values.
32  */
