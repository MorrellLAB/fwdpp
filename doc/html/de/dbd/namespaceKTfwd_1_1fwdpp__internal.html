Mechanics of segregation, recombination, and mutation for multi-locus API This template declaratiopn is messy due to the FWDPP_UNIT_TESTING symbol.

The reason is that I want to be able to unit test the recombination portion of this code in isolation, w/o having to do anything involving mutation policies.

Thus, the relevant unit tests define FWDPP_UNIT_TESTING so that the mutation-related stuff is skipped during compilation.
